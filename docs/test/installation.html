Installation

System Prerequisites

This is a full node experience, so all caveats of running a full node apply.

In particular, for mainnet you will need:

  • 24 GB RAM (UTxO set is still in memory)
  • 250 GB disk space (Transaction history is stored in full)
  • 4 CPU cores (for syncing)

But for preprod you can get away with:

  • 2 GB RAM
  • 5 GB disk space
  • 2 CPU cores

CLI installation instructions

You will need some tools to follow the installation:

  • jq
  • wget
  • tar
  • screen
  • a working browser

Nixers can run a shell with all the tools:

nix-shell -p jq wget gnutar screen

or

nix shell nixpkgs#jq nixpkgs#wget nixpkgs#gnutar nixpkgs#screen

We are going to use preprod network in these instructions. Small changes are needed to run on other networks.

Create a directory for the node state or use the current directory:

NODE_DB=${NODE_DB:-$(pwd)/node-db}
export NODE_DB
mkdir -p "$NODE_DB"
rm -rf "$NODE_DB/preprod"

Now you have 2 choices on how to proceed

Using a package for your system.

Not all platforms are supported, but we have packages for linux64, macos-silicon, and macos-intel.

  • Using a platform package. As we do not ship for Windows, you will have to use docker solution.

Using a docker image

The docker image is build through the nix package manager and has no base image.

  • Using a docker image

Unpack the wallet package

Once the WALLET_PACKAGE is set, you can unpack the wallet package.

tar xvzf "$WALLET_PACKAGE" > /dev/null
WALLET_DIR=$(pwd)
export WALLET_DIR
export PATH=$WALLET_DIR:$PATH

Node database initialization

It's recommended to retrieve the node state from Mithril to avoid long waiting times.

Mithril node database setup

Screen and cleanup

We are going to use screen to keep the processes running in the background, but you can use any other tool you like.

If you are using this instructions as a script you should install cleanups to ensure the screen sessions are closed.

Otherwise, you can skip the next step and cleanup the screen sessions manually at the end.

cleanup() {
    screen -S "$NODE_SESSION" -X quit || true
    screen -S "$WALLET_SESSION" -X quit || true
}
trap cleanup EXIT
trap cleanup SIGINT

Start the node

Now you can start the node that comes bundled with the wallet.

NODE_SESSION="node-session-$(shuf -i 1000000-9999999 -n 1)"
NODE_CONFIGS="$WALLET_DIR/configs/preprod"
export NODE_CONFIGS
screen -dmS "$NODE_SESSION" cardano-node run \
    --config "$NODE_CONFIGS/config.json" \
    --database-path "$NODE_DB/preprod" \
    --socket-path "$NODE_DB/preprod/node.socket" \
    --topology "$NODE_CONFIGS/topology.json"

You can observe the node logs.

screen -r "$NODE_SESSION"

Detach from the screen with Ctrl-a d.

Start the wallet

Now you can start the wallet.

WALLET_SESSION="wallet-session-$(shuf -i 1000000-9999999 -n 1)"
DEPOSIT_PORT=$(shuf -i 1024-65000 -n 1)
export DEPOSIT_PORT
LEGACY_PORT=$(shuf -i 1024-65000 -n 1)
screen -dmS "$WALLET_SESSION" cardano-wallet serve \
    --node-socket "$NODE_DB/preprod/node.socket" \
    --testnet "$NODE_CONFIGS/byron-genesis.json" \
    --ui-deposit-port "$DEPOSIT_PORT" \
    --port "$LEGACY_PORT"

ATM the legacy port is required even if we are not using it. Once the deposit wallet is extracted from the wallet, the legacy port will be removed.

You can observe the wallet logs with:

screen -r "$WALLET_SESSION"

Detach from the screen with Ctrl-a d.

Test the web UI

Use curl to check the UI is up:

sleep 10
curl -s "http://localhost:$DEPOSIT_PORT" > /dev/null

Open the web UI with your browser

Now you can connect to the web UI with:

xdg-open "http://localhost:$DEPOSIT_PORT"

Manual cleanup

Clean up the screen sessions with:

screen -S "$NODE_SESSION" -X quit
screen -S "$WALLET_SESSION" -X quit
